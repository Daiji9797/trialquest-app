# TrialQuest 事業企画書

---

## 1. エグゼクティブサマリ

**何を**：MBTI 性格診断とパーソナル回答をもとに、AI エージェントが「5 つの大陸」の視点から最適なクエスト（行動提案）を生成するパーソナライズ型 AI コーチング Web アプリ

**誰に**：「自分がどんな行動をとるべきかわからない」「行動が続かない」成人ユーザー全般（社会人・学生）

**どう解決**：MBTI 軸で性格を把握し、健康・知識・人間関係・行動・創造の 5 領域から AI エージェントがパーソナライズされた具体的クエストを提示。提案の実施状況を管理して行動を継続させる

**どう儲ける**：基本無料 ＋ 高度なエージェント機能・履歴分析・企業向けプランで収益化

**一言**：**「あなたの性格に合った、今日すべき行動をAIが設計する」**

---

## 2. 課題（Why / Who）

**対象ユーザー**
- 行動したいが何から手をつけるべきかわからない社会人・学生
- 目標はあるが自分の性格・強みを活かした行動が取れていない人
- コーチングや自己啓発サービスに興味があるがコスト・敷居が高い人

**解決すべき課題**
- 汎用的なアドバイスは自分に刺さらない → **性格（MBTI）に特化した提案がない**
- 複数領域（健康・仕事・人間関係等）を横断して考えられるサービスがない
- 行動提案を受けても記録・管理できず継続できない
- 専門コーチへのアクセスはコストが高い

---

## 3. 解決策（How）

1. **MBTI パーソナライズ**：EI / SN / TF / JP の 4 軸をスライダーで入力。過去の診断履歴も保持し、変化を追跡可能
2. **5 大陸 AI エージェント**：健康・知識・人間関係・行動・創造の各専門 AI エージェントが、MBTI と回答内容に基づいた深いクエストを生成
3. **コンダクター（統合エージェント）**：複数領域を選択すると council-agent が各専門エージェントを統合し、「まず何から取り組むべきか」を提示
4. **提案ステータス管理**：「実施する / 完了 / 削除」でクエストの実行状況を管理。LocalStorage + CosmosDB で永続保存
5. **興味コンテキスト入力**：「今興味があること」を自由記述で追加し、よりタイムリーな提案を取得

## 4. 価値提案

**「5人のAI専門家があなたの性格を理解して、今日の行動を設計する」**

| 価値 | 内容 |
|------|------|
| パーソナライズ | MBTI × 回答 × 興味の3軸で個別最適化 |
| 専門性 | 健康・知識・人間関係・行動・創造の各専門エージェント |
| 継続性 | 提案の実施管理 + 診断履歴の蓄積 |
| 統合性 | コンダクターが複数領域を横断して優先順位を提示 |
| 手軽さ | GitHub アカウントで即利用開始 |

## 5. 競合比較

| 観点 | 汎用 AI チャット (ChatGPT等) | 目標管理アプリ (Notion等) | **TrialQuest** |
|------|------|------|------|
| 性格ベースの提案 | なし | なし | **MBTI連動** |
| 専門領域の深さ | 汎用のみ | なし | **5大陸専門エージェント** |
| 複数領域の統合 | 手動で依頼 | なし | **コンダクター自動統合** |
| 提案の継続管理 | なし | 手動 | **実施/完了/削除ステータス** |
| 診断履歴の蓄積 | なし | 手動 | **MBTI履歴自動保存** |
| コスト | 月額課金 | 月額課金 | **基本無料** |

## 6. 収益モデル

**無料プラン**
- MBTI 診断・履歴保存（直近 N 件）
- 大陸選択 1 つまで（singleAgent）
- クエスト提案・ステータス管理

**有料プラン（個人向け）**
- 全大陸・コンダクター機能の無制限利用
- 診断履歴の無制限保存・分析
- 回答履歴のダウンロード

**法人向けプラン**
- 新人研修・1on1 面談の補助ツールとして導入
- 管理者ダッシュボード（チームメンバーの行動傾向可視化）
- カスタムエージェント追加（企業独自の行動指針を反映）

## 7. 市場性

**対象市場**
- 自己啓発・パーソナル AI コーチング・キャリア支援領域
- MBTI 関連市場（SNS での MBTI 人気は継続中）
- 企業向け人財開発・研修市場

**強み**
- MBTI × 生成 AI の組み合わせは既存サービスに前例が少ない
- AI Foundry によるエージェントアーキテクチャは品質・拡張性が高い

## 8. Go-To-Market

**初期ターゲット**
- MBTI に関心のある 20〜35 歳のビジネスパーソン・学生
- 自己成長・習慣化に意欲があるが何をすべきかわからない層

**獲得チャネル**
- MBTI 関連 SNS コミュニティ（Twitter/X・Instagram）
- キャリア支援プラットフォームとの連携
- 企業研修・1on1 支援ツールとしての B2B 展開

**ローンチ形態**
- GitHub OAuth で即座に利用開始（登録フォーム不要）
- 招待制ベータ → 段階的一般公開

## 11. リスクと対策

| リスク | 内容 | 対策 |
|--------|------|------|
| AI 品質 | エージェントの回答がユーザーに刺さらない | Foundry ポータルでプロンプトを継続改善（コード変更不要） |
| セキュリティ | ユーザーデータの漏洩 | GitHub OAuth + SWA 認証ゲート + CosmosDB userId 分離 |
| コスト | Azure/AI Foundry の従量課金増大 | 利用上限設定・有料転換で費用回収 |
| 依存リスク | Azure AI Foundry の仕様変更 | 抽象化レイヤー（Function App）でエンドポイントを隠蔽 |
| 継続率低下 | クエストをこなして終わりになる | 提案の更新サイクル・季節性コンテンツ・通知機能で再想起 |

## 12. 技術構成（付録）

```
ブラウザ（React SPA）
  └── GitHub OAuth（SWA 組み込み）
Azure Static Web Apps
  ├── フロントエンド（React）
  └── SWA Managed Functions（CosmosDB 接続）
      └── agentSuggestions プロキシ → [function key]
Azure Functions（Managed Identity）
  ├── 1大陸選択時：Foundry Agents API で instructions 取得（5分キャッシュ）
  │                → Chat Completions で実行
  └── 2大陸以上：council-agent Application（マルチエージェント統合）
Azure AI Foundry
  ├── 5大陸エージェント（health / knowledge / relationship / action / creation）
  └── council-agent（統合コンダクター）
Azure Cosmos DB
  └── userProfile / questions / answers
```

**主要技術**

| 項目 | 内容 |
|------|------|
| フロントエンド | React（CRA）、GitHub Pages 不使用・SWA でホスト |
| バックエンド | Azure Functions v4（Node 20、CommonJS） |
| AI 基盤 | Azure AI Foundry（gpt-4o-mini）、Managed Identity 認証 |
| データベース | Azure Cosmos DB（NoSQL） |
| 認証 | Azure Static Web Apps 組み込み GitHub OAuth |
| セキュリティ | SWA 認証ゲート + Function App function key + Managed Identity |
